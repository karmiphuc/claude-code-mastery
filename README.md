## 🎯 Who Is This For?

- **Complete beginners** who've never used a terminal before
- **Claude users** who want to upgrade to Claude Code
- **Developers** learning terminal-based AI workflows

## 📋 What You'll Learn

### Module 0: Terminal Orientation
- ✅ What Claude Code is and why it uses a terminal
- ✅ Understanding the TUI layout (Response Area, Input Area, Status Bar)
- ✅ Key keyboard shortcuts (Escape, Enter, Shift+Tab)
- ✅ Slash commands basics (/clear, /help)
- ✅ Permission prompts and how to respond

### Module 1: Foundations
- ✅ Starting Claude Code (setup already done via company script)
- ✅ First-run configuration
- ✅ Your first conversation
- ✅ Non-coding wins (file organization, research)
- ✅ Essential slash commands

### Module 2: Core Skills
- ✅ Full keyboard mastery
- ✅ Understanding Auto, Plan, and Edit modes
- ✅ Plan Mode deep dive
- ✅ Message queuing for batch tasks
- ✅ Context management and `/clear`
- ✅ File referencing techniques

### Module 3: Mastery
- ✅ CLAUDE.md configuration
- ✅ Memory system (#hashtags)
- ✅ Custom slash commands
- ✅ Hooks for automation
- ✅ GitHub integration
- ✅ Power user tips and workflows

### Bonus: 10 Productivity Wins (Non-Engineering)

- ✅ The "Digital Janitor" (File Organization)
- ✅ The "Excel Killer" (Data Analysis)
- ✅ The "Local Wikipedia" (Knowledge Management)
- ✅ The "Ghostwriter" (Content Repurposing)

**Part 2: Power Workflows**
- ✅ The "Project Architect" (Instant Setup)

## 📝 How to Use This Curriculum

### General Approach

1. **Read each module in order** (first time through)
2. **Do all exercises** — Reading isn't enough
3. **Check off progress** — Use the tracker
4. **Review weak areas** — Self-assessment guides you
5. **Practice daily** — Even 15 minutes helps

### Time Commitment

| Pace | Duration | Daily Time |
|------|----------|------------|
| Intensive | 1 week | 2-3 hours/day |
| Standard | 2 weeks | 1-1.5 hours/day |
| Relaxed | 4 weeks | 30-45 min/day |

### Tips for Success

- ✅ **Complete exercises** — Don't just read
- ✅ **Use the sandbox folder** — Keep experiments safe
- ✅ **Commit frequently** — When working on real code
- ✅ **Take notes** — Write down what works for you
- ✅ **Build habits** — `/clear` before new tasks, Escape to stop

- [ ] Navigate Claude Code's TUI confidently

- [ ] Maintain a useful CLAUDE.md
